Introduction to DBMS Terms


Data

Data refers to raw facts and figures that can be processed and interpreted to gain information.

Example: Numeric values, text strings, dates, images, etc.


Table

In a database context, a table is a structured collection of data organized into rows (records) and columns (fields), where each row represents a unique record and each column represents a data attribute.

Example: In a "Students" table, each row could represent a different student, and columns could include attributes like student ID, name, and date of birth.


Information

Information is processed and organized data that has context and meaning. It is the result of interpreting and analyzing data to make it useful and meaningful for decision-making.

Example: Reports, summaries, statistics, etc.


Metadata

Metadata is data that provides information about other data. It describes the structure, content, and context of data elements within a database or data repository.

Example: Descriptions of data fields, data types, constraints, etc.


DBMS (Database Management System)

A DBMS is a software system that enables users to create, manage, and manipulate databases, providing functions for data storage, retrieval, modification, and security.

Example: MySQL, PostgreSQL, Oracle Database, SQL Server


Data Dictionary

A data dictionary is a centralized repository of metadata that contains information about the data within a database. It describes the structure, relationships, and attributes of data elements.

Example: In a data dictionary, you might find descriptions of tables, columns, data types, and constraints used in a database.


Database

A database is a structured collection of data organized and stored electronically. It allows for efficient data manipulation, retrieval, and management.

Example: An online store might have a database containing tables for customers, orders, products, and reviews.


Tuple (Row)

A tuple refers to a single record or row in a table, containing a set of related data fields that collectively represent an entity.

Example: In an "Employees" table, each tuple (row) might contain information about an individual employee, such as employee ID, name, department, and salary.


Field (Column)

A field, also known as a column, is a single piece of data within a table. It represents a specific attribute or characteristic of the entities that the table represents.

Example: In a "Books" table, fields (columns) could include book ID, title, author, publisher, and publication year.


Domain

Domain refers to the permissible range of values for a specific data attribute or field in a database, defining the valid data types, formats, and constraints that govern data input and integrity.

Example: In a database, a domain for a "Date of Birth" field might specify that the values must be in the format YYYY-MM-DD and must correspond to valid dates.


Constraint

A constraint is a rule or condition applied to data within a database to maintain the integrity, accuracy, and reliability of the data. Examples include uniqueness, foreign key, and check constraints.

Example: A "Unique Constraint" ensures that values in a particular column (or combination of columns) are unique across all rows in a table, preventing duplicate entries.


Key

A key is a field or combination of fields that uniquely identifies a record within a table. Examples include primary keys, candidate keys, and foreign keys.

Example: A "Primary Key" in a "Students" table might be the Student ID, ensuring each student record has a unique identifier.
